## Key Insights

1. **Docker anonymous volumes are fragile under OOM conditions** — consider explicit named volumes in `docker-compose.dev.yml` for node_modules and vendor to make recovery deterministic.

2. **The pnpm Docker workaround is now a known pattern** — always: edit `package.json` directly, then `pnpm install --store-dir /root/.local/share/pnpm/store/v10 --no-frozen-lockfile` inside container. Never `pnpm add` inside Docker.

3. **BusyBox `nc -z` is not reliable for healthchecks** — use `curl -sf http://localhost:PORT/up` for HTTP services, `nc -w1 localhost PORT < /dev/null` for raw TCP.

4. **`ApiResponse::error()` defaults to HTTP 400** — named constructors (401/403/404/422) must be added before any Epic 2 auth endpoints are written.
